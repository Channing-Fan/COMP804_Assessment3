Name Web Attacks

Malware attacks

Malware refers to malicious software designed to exploit devices for the attacker's benefit, often using stealthy techniques to avoid detection and gain unauthorised access.

Ransomware encrypts victims' files and demands payment for their release. Rootkits open backdoors for further attacks. Trojans that masquerade as harmless programs, and spyware that secretly monitors and steals sensitive information.

Learn more

Ransomware attack

Ransomware attacks typically occur when a user visits a malicious website or opens a harmful email attachment, leading to the encryption of important files on the infected device.

Attackers then demand a ransom for the decryption key needed to unlock these files, and in some cases, they also threaten to sell or leak stolen data to increase pressure on the victims.

Learn more

Password attacks

Passwords remain the most common, yet vulnerable, method of authentication, making them a prime target for attackers seeking unauthorised access to systems and data.

Attackers use various methods to acquire passwords, including brute-force and dictionary attacks that guess passwords, social engineering tactics to trick users, and keylogging software that captures keystrokes.

Additionally, they may use password sniffers on networks or steal password databases directly from organisations.

Learn more

DDoS attack

A distributed denial-of-service (DDoS) attack floods a target server or network with excessive traffic from numerous compromised devices, aiming to slow it down or crash it, denying service to legitimate users.

These attacks are often motivated by political or ideological reasons, but some are conducted for ransom, with attackers threatening to launch a DDoS attack unless payment is made.

The use of AI is increasingly enhancing the effectiveness of these attacks, making them more sophisticated and harder to defend against.

Learn more

Phishing

Phishing is a type of attack where an attacker pretends to be a trusted entity to deceive a victim into revealing sensitive information like passwords or credit card details.

These attacks, often carried out via email, text messages, or phone calls, are based on social engineering and are highly effective and easy to execute.

Variants include spear phishing, which targets specific individuals or companies, and whaling, which focuses on high-level executives.

Learn more

SQL injection

SQL injection attacks target websites that use databases by manipulating SQL queries to perform unauthorised actions.

These attacks can alter, delete, or steal sensitive data stored in the database, including personal information and business details.

As a result, they pose a significant risk to the security of database-driven websites.

Learn more

Cross-site scripting

Cross-site scripting (XSS) is a type of injection attack where an attacker injects malicious scripts into a legitimate website, which are then executed in the browsers of unsuspecting users.

This allows the attacker to perform actions such as stealing session cookies, distributing malware, or defacing websites.

Often, XSS is combined with social engineering tactics like phishing to increase its effectiveness.

Learn more

Man-in-the-middle attacks

A man-in-the-middle (MitM) attack involves an attacker secretly intercepting and controlling communications between two unsuspecting parties, such as an end user and a web application.

The attacker can read, alter, and forward messages, potentially capturing sensitive information like login credentials and financial details.

These attacks are often aimed at online banking and e-commerce users, sometimes initiated through phishing emails that install malware on the victim's device.

Learn more

URL interpretation/URL poisoning

This type of attack involves an attacker modifying a URL to gain unauthorised access to information or resources on a website.

By altering parameters in the URL, such as user identifiers, the attacker can potentially access other users' data if the website lacks proper authorisation checks.

This vulnerability, often due to insecure direct object references, allows the attacker to exploit the site's failure to verify user identities correctly.

Learn more

DNS spoofing

DNS spoofing attacks manipulate the Domain Name System (DNS) to redirect users to malicious websites by altering stored IP addresses.

These fake sites closely mimic legitimate ones, deceiving users into providing sensitive information such as login credentials.

The attackers then use this information to access the victims' real accounts on the targeted sites.

Learn more
